AIFIX 데이터 관리 탭 – 조회 결과 스키마 가시성 개선 및 상태 시나리오 추가

현재 데이터 관리 탭의 조회 결과 영역은 구매 관점과 PCF 관점이 통합되었다.

하지만 현재 UI에는 다음 문제가 존재한다.

1️⃣ 협력사 Row에 컬럼 헤더가 없어 각 데이터의 의미(스키마)를 이해하기 어렵다.
2️⃣ 상태값이 하나의 영역에 묶여 있어 데이터 제출 상태 / 검증 상태 / 리스크 상태를 구분하기 어렵다.
3️⃣ 숫자 데이터의 단위와 의미가 명확하게 드러나지 않는다.

따라서 조회 결과 영역의 스키마 가시성을 개선하도록 UI를 재구성하라.

1️⃣ 프로젝트 Accordion 구조 유지

현재 구조는 유지한다.

프로젝트 카드
→ 클릭 시
→ 협력사 데이터 Row 표시

예

SCN-2026-01
BMW iX5 배터리 공급

SCN-2026-02
Mercedes EQS 차세대 배터리

SCN-2026-03
Audi Q6 e-tron ESS

프로젝트가 닫혀 있으면 협력사 데이터는 보이지 않는다.
프로젝트를 열면 협력사 데이터 테이블이 표시된다.

2️⃣ 협력사 데이터 테이블 헤더 추가

프로젝트가 펼쳐졌을 때 협력사 데이터 테이블의 컬럼 헤더를 표시한다.

3️⃣ 컬럼 그룹 구조 적용

헤더는 다음과 같이 그룹 구조로 구성한다.

기업 정보
→ Tier / 회사명 / 국가 / 회사 유형

공급 데이터
→ 납품량 / 원재료 투입량 / 제품 유형

탄소 데이터
→ 배출계수 DB / PCF 결과 / 배출 강도

상태
→ 데이터 제출 상태 / 검증 상태 / 리스크 상태

액션
→ 상세보기

4️⃣ 상태 컬럼 분리

현재 상태 배지는 하나의 영역에 묶여 있다.

이를 다음 3개의 컬럼으로 분리한다.

데이터 제출 상태
예: 미제출 / 제출완료

검증 상태
예: 미검증 / 검증완료

리스크 상태
예: 낮음 / 보통 / 높음

5️⃣ PCF 산정 상태 시나리오 표현

협력사 데이터는 다음 두 가지 상태가 존재할 수 있다.

1️⃣ PCF 산정 완료

예

Tier0 | 삼성SDI | South Korea | Operator
납품량: 50,000 kg
투입량: 30,000 kg
제품 유형: 배터리 모듈
배출계수 DB: Internal DB v3.2
PCF 결과: 32,420 kg CO₂e
배출 강도: 0.648 kg CO₂e/kg

데이터 제출 상태: 제출완료
검증 상태: 검증완료
리스크: 낮음

2️⃣ PCF 산정 미완료

예

Tier2 | 셀테크 | China | Supplier
납품량: 20,000 kg
투입량: 15,000 kg
제품 유형: 리튬이온 셀
배출계수 DB: DEFRA 2023

PCF 결과: 미산정
배출 강도: -

데이터 제출 상태: 제출완료
검증 상태: 미검증
리스크: 보통

6️⃣ 숫자 컬럼 단위 표시

모든 수치 컬럼에는 단위를 명확히 표시한다.

납품량 → kg
투입량 → kg
PCF 결과 → kg CO₂e
배출 강도 → kg CO₂e/kg

참고

프로젝트 → 협력사 → Tier 공급망
