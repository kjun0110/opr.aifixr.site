AIFIX – 협력사 데이터 관리 “분기 기반 조회 구조” 개편

너는 제조업 ESG·PCF 통합 SaaS AIFIX의 시니어 프로덕트 디자이너다.

현재 협력사 데이터 관리 화면의 조회 조건에는 기간 선택 기능이 없다.

협력사는 매월 데이터 입력이 현실적으로 어렵기 때문에
입력 단위는 “분기(Quarter)”로 운영한다.

이를 반영하여 조회 조건 및 테이블 구조를 전면 개편하라.

✅ 1️⃣ 조회 조건에 “분기 선택” 추가

조회 조건 영역에 아래 항목 추가:

연도 선택 ▼
분기 선택 ▼

또는 단일 Range 선택 방식:

[ 2026년 Q1 ~ 2026년 Q3 ]
✅ 2️⃣ 분기 선택 방식

연도 드롭다운

분기 드롭다운 (Q1 / Q2 / Q3 / Q4)

범위 선택 가능

초기값: 최근 완료 분기

✅ 3️⃣ 협력사 입력 단위

협력사는 분기 단위로만 입력한다.

월 입력 UI는 제공하지 않는다.

✅ 4️⃣ 고객사가 월 단위 요청하는 경우 대비

예: 2월~10월 요청

이를 처리하기 위해

테이블에 반드시 아래 두 값을 함께 표시한다:

분기 합산 값

월 추정치 값

월 추정치 계산 기준:

분기 값 ÷ 3

(고급화 가능)

🔵 5️⃣ PCF 관점 테이블 구조

컬럼 구성:

| Tier | 회사명 | 분기 배출량 (tCO₂e) | 월 추정 배출량 | DQR | 계산 가능 | 상세보기 |

설명:

분기 배출량 = 협력사 입력값

월 추정 배출량 = 자동 계산

분기 범위 선택 시 합산 계산

🟣 6️⃣ 구매 관점 테이블 구조

컬럼 구성:

| Tier | 회사명 | 분기 납품량 | 월 평균 납품량 | 인증 상태 | 리스크 등급 | 제출 상태 |

설명:

분기 납품량 = 분기 합계

월 평균 납품량 = 분기값 ÷ 3

리스크 색상 뱃지 유지

✅ 7️⃣ UI 차이점 명확히 반영

구매 관점 / PCF 관점 전환 시:

조회 조건은 동일 (분기 기반)

테이블 컬럼은 완전히 다르게 구성

계산 필드 종류도 다름

✅ 8️⃣ UX 원칙

협력사 입력은 단순화

원청 조회는 세분화

월 추정치는 “연한 색상”으로 표시 (보조값 느낌)
