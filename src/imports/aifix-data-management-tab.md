AIFIX – 원청 구매직무 전용 [데이터 관리] 탭 설계

너는 B2B 제조업 ESG·공급망 관리 SaaS AIFIX의 시니어 프로덕트 디자이너다.

(PCF 담당자 화면은 포함하지 않는다.)

0️⃣ 화면 목적 (Goal)

구매직무가:

자사 데이터를 관리하고

협력사로부터 제출받은 데이터를 열람·수정·검증하고

엑셀 기반 대량 업로드/다운로드를 수행하며

키인 입력 시 실시간 Validation을 통해 오류를 바로 수정할 수 있도록

설계한다.

1️⃣ 상위 구조

[데이터 관리] 탭 내부에 하위 탭 2개를 둔다.

🔹 자사데이터 관리

🔹 협력사 데이터 관리

상단에 세그먼트형 탭 UI로 표현한다.

2️⃣ 자사데이터 관리 화면 설계
🎯 목적

원청 구매직무가 ERP/기타 시스템과 인터페이스하여 들어온 자사 데이터를 관리하고,
부족한 부분은 직접 키인으로 보완한다.

2-1 상단 컨트롤 영역

우측 상단에 다음 버튼 배치:

[엑셀 다운로드]

[엑셀 업로드]

[신규 추가]

[저장]

엑셀 다운로드:

현재 필터 조건 기반 다운로드

엑셀 업로드:

파일 업로드 모달

성공/실패 건수 표시

오류 항목 미리보기 제공

2-2 데이터 테이블 구성

테이블은 수정 가능(editable table) 구조.

컬럼 예시 (임시 구조):

고객사

프로젝트

제품

사업장

공장

공정

자사 BOM ID

BOM 버전

자재코드

자재명

내부 관리 상태

수정일

수정자

각 셀은 클릭 시 인라인 수정 가능.

2-3 키인 입력 시 실시간 Validation

Validation UX 반드시 구현:

형식 오류 시 해당 셀 빨간 테두리

셀 하단에 작은 회색 설명 텍스트

예:

“사업자등록번호는 숫자 10자리여야 합니다.”

“국가코드는 ISO 2자리 형식으로 입력하세요.”

저장 버튼 클릭 시:

오류가 하나라도 있으면 저장 불가

상단에 오류 요약 배너 표시

3️⃣ 협력사 데이터 관리 화면 설계
🎯 목적

특정 제품 + 특정 공급망 선택 시,
해당 공급망에 포함된 1차 및 하위차사들이 말아올려 제출한 정보를
구매직무가 한눈에 열람·수정·검증할 수 있도록 한다.

3-1 상단 필터 영역

드롭다운:

▽ 제품

▽ 공급망 버전

선택 후 아래에 해당 공급망 데이터 표시

3-2 데이터 표시 방식
기본 구조: Tier 구조 기반 그룹 테이블

화면은 다음 구조로 설계:

Tier 1 (카드 또는 그룹 헤더)
└ Tier 2
└ Tier 3

트리 구조 테이블 형태로 표현한다.
각 Tier는 접기/펼치기 가능.

3-3 구매직무가 확인하고 싶어할 핵심 컬럼만 표시
A) 기업 식별 요약

협력사 회사명 (영문 병기)

차수(Tier)

공급자 유형

국가

플랜트명

Tax ID (마스킹)

DUNS

B) 규제 및 리스크 핵심 정보

FEOC 여부

RMI Smelter 여부

인증 광산 여부

리스크 상태 아이콘

C) 제출 상태

제출 상태

마지막 제출일

담당자 이름

승인 여부

D) 원청 BOM 연계 표시 (중요)

해당 공급망과 연결된:

원청 BOM 목록 (태그)

BOM 버전

동일 공급망이라도 여러 BOM 연결 가능하도록 태그 UI 구성

3-4 행 단위 액션

각 협력사 행 오른쪽:

[상세 보기]

[수정]

[검증 요청]

[이력 보기]

3-5 수정 UX

수정 모드 진입 시:

인라인 편집 가능

실시간 Validation 동일 적용

오류 시 빨간 테두리 + 하단 설명 텍스트

4️⃣ 공통 Validation UX (자사/협력사 공통)

모든 키인 필드에 대해:

자리수 검증

형식 검증 (이메일, 숫자, ISO코드 등)

필수값 누락 검증

오류 UX:

셀 빨간 보더

하단 12~14px 안내문

상단 에러 요약 배너

5️⃣ Empty / Loading 상태

공급망 미선택 시: “제품과 공급망을 선택하세요.”

데이터 없음: 안내 카드

로딩: 스켈레톤 UI

6️⃣ 최종 산출물

고해상도 데스크톱(1440px 기준) 화면 2개 설계:

자사데이터 관리 화면

협력사 데이터 관리 화면

엔터프라이즈 B2B SaaS 수준의 정돈된 UI로 설계하라.
