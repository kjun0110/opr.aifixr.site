AIFIX – 협력사 데이터 관리 탭 구조 전면 개편

너는 제조업 공급망·PCF 통합 SaaS AIFIX의 시니어 프로덕트 디자이너다.

현재 “데이터 관리 → 협력사 데이터 관리” 화면은
필터를 바꿔도 동일한 결과가 나열되는 정적 테이블 구조다.

이를 실제 공급망 트리 기반 동적 화면 구조로 전면 개편하라.

✅ 1️⃣ 상단 필터 구조 재설계

좌측 필터 패널을 다음 구조로 변경하라.

기본 드롭다운 구성
▽ 제품 선택
▽ 공급망 선택
▽ 차수 선택
▽ 제출 상태 (추가)
▽ 국가 (추가)
필터 동작 원칙

제품 선택 → 해당 제품에 연결된 공급망만 노출

공급망 선택 → 해당 공급망에 연결된 1차 협력사 기준으로 트리 생성

차수 선택 → 해당 차수까지만 트리 노출 (예: 2차까지 보기)

제출 상태 선택 → 승인/검토중/반려 필터 적용

국가 선택 → 해당 국가 협력사만 표시

👉 필터 조합에 따라 결과가 달라져야 한다
👉 모든 필터는 실제 데이터 구조에 영향을 주는 것으로 설계하라

✅ 2️⃣ 협력사 데이터 영역 – 트리형 구조로 변경

현재 단순 나열 테이블을 제거한다.

🔵 기본 구조

공급망 선택 시:

1차 협력사를 최상단 노드로 표시
▶ Tier 1 – 한국배터리
    ▷ Tier 2 – 셀테크
        ▷ Tier 3 – 리튬소재
    ▷ Tier 2 – 글로벌소재
설계 원칙

원청(0차)은 이 화면에서 표시하지 않는다

항상 1차가 최상위 노드

각 노드는 접기/펼치기 가능

기본은 1차만 펼침

하위 차수는 접힌 상태

상단에 관점 토글 유지:

✅ 4️⃣ 차수 드롭다운 동작 방식

차수 선택:

“1차” → 1차 노드만 노출

“2차” → 1~2차까지만 펼침

“전체” → 전체 트리 구조 표시

✅ 5️⃣ 공급망 드롭다운 동작

공급망 선택 시:

해당 공급망에 연결된 1차만 노출

다른 공급망 협력사는 완전 제외

결과 영역 전체 리렌더링

✅ 6️⃣ 디자인 기준

디자인은 네 번째 이미지의 프로젝트 트리형 구조와 동일한 패턴을 사용하라.

그룹 단위 카드

차수 배지 표시 (Tier 1 / Tier 2)

좌측에 트리 인디케이터

상태 아이콘 컬러 구분

✅ 7️⃣ 경우의 수 확장

현재는 한 가지 데이터 세트만 보여주는 구조다.

이를 다음과 같이 확장하라:

동일 제품 내 다수 공급망 존재 가능

동일 공급망 내 다수 1차 존재 가능

1차가 여러 2차를 가질 수 있음

2차가 여러 3차를 가질 수 있음

트리 구조는 재귀적으로 설계하라.

✅ 8️⃣ UX 개선 포인트

1차 노드에 “전체 펼치기” 버튼

“모두 접기” 버튼

노드 클릭 시 우측 슬라이드 상세 패널
