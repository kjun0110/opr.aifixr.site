AIFIX – 프로젝트 관리 탭 역할 분리 및 구조 정제

너는 제조업 대기업 대상 ESG/공급망 관리 SaaS AIFIX의 시니어 프로덕트 디자이너다.

현재 프로젝트 관리 탭에는
공급망 연결 관리 기능과 PCF 계산 실행 기능이 혼재되어 있다.

프로젝트 관리 탭을 **구매 관점 전용 “맥락 정의 화면”**으로 재정비하라.

✅ 1️⃣ PCF 실행 기능 제거 (중요)
🎯 목적

구매와 PCF 직무 역할 분리

🔥 수정 지시
프로젝트 상세 화면에서:

“PCF 계산 실행” 버튼 제거

계산 트리거 UI 제거

계산 버튼 색상 강조 요소 제거

대신 다음 정보는 유지 (조회 전용)

각 제품 카드 하단에:

PCF 계산 상태 (미계산 / 일부 완료 / 완료)

최근 계산일

계산 결과 존재 여부 아이콘

실행은 불가, 조회만 가능하도록 설계.

버튼 대체

기존 계산 버튼 위치에:

[PCF 탭에서 계산하기] 링크 버튼 추가

클릭 시 PCF 관점 탭으로 이동.

✅ 2️⃣ 프로젝트 목록 화면 정제

현재 컬럼은 유지하되 다음 원칙 적용:

공급망 버전 직접 표시 금지

M-BOM 버전 직접 표시 금지

제품 집계 기반 상태만 표현

상태 표현 로직

프로젝트 상태는 제품 집계 기반으로 표시:

제품 미연결

구조 미연결 제품 존재

M-BOM 미선택 제품 존재

PCF 일부 완료

PCF 완료

단일 공급망 기반 상태 표현 금지.

✅ 3️⃣ 고객사 → 프로젝트 계층 정제

현재 구조는:

고객사 → 프로젝트

이 구조는 유지한다.

단, 프로젝트 생성 모달에 다음 필드 추가:

납품 법인

납품 지역

납품 공장

생산 거점 (선택)

고객사 아래에서 여러 납품 단위를 표현할 수 있도록 한다.

예시:

고객사: BMW
프로젝트:

BMW iX5 – Germany Plant

BMW iX5 – US Plant

별도 “지사 테이블”은 만들지 않는다.

✅ 4️⃣ 프로젝트 생성 모달 개선

현재 필드:

프로젝트 코드

프로젝트명

고객사 선택

납품 시작/종료일

추가 필드:

납품 지역 (드롭다운)

납품 공장명

법인명 (선택)

✅ 5️⃣ 상단 안내 문구 수정

현재 문구를 다음으로 변경:

프로젝트 관리 탭은 고객사 납품 맥락을 정의하고,
제품별로 공급망 버전과 M-BOM을 연결하는 화면입니다.
PCF 계산 실행은 PCF 관점 탭에서 수행합니다.

✅ 6️⃣ 권한 구조 반영

구매 관점:

프로젝트 생성/수정 가능

제품 연결 가능

공급망 선택 가능

M-BOM 선택 가능

PCF 실행 불가

PCF 관점:

프로젝트 조회 가능

PCF 실행 가능

공급망 수정 불가

🎯 최종 설계 목표

프로젝트 관리 탭은:

구조 정의 X

계산 실행 X

연결 정의 O

공급망은 1계층 자산
프로젝트는 2계층 맥락
PCF는 3계층 실행

역할이 UI에서 명확히 구분되어야 한다.
