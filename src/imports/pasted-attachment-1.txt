너는 제조업 ESG/공급망/PCF SaaS AIFIX의 시니어 프로덕트 디자이너다.
현재 시스템은 상단 메인 네비게이션과 우측 상단 “업무 관점 토글(구매 관점 / PCF 관점)”이 존재한다.

이번 작업 목표는 다음 3가지이다.

0) 목표(Goal)

PCF 관점일 때만 메인 네비게이션에 [PCF 산정] 탭이 보이도록 한다.

[PCF 산정] 탭에서 원청 + 협력사 데이터를 합쳐 최종 PCF를 산정하는 핵심 화면을 설계한다.

산정 결과에 대해 공급망 구성(버전) 단위로 관리되며, 모든 협력사/항목별로 사용한 배출계수(EF) 출처와 ID를 추적할 수 있어야 한다.

✅ 1) 메인 네비게이션 수정 (조건부 노출)
1-1. 탭 추가

메인 네비게이션에 다음 탭을 추가한다.

[PCF 산정]

1-2. 조건부 노출 규칙

사용자가 우측 상단 토글에서 PCF 관점을 선택했을 때만

메인 네비게이션에 [PCF 산정] 탭이 노출

구매 관점일 때는

[PCF 산정] 탭이 숨겨진다

전환 시 UX:

PCF 관점으로 전환하면 [PCF 산정]이 나타나며, 강조색(보라)로 표시 가능

구매 관점으로 전환하면 탭이 사라지고, 현재 탭이 PCF 산정이면 자동으로 “데이터 조회”로 이동

✅ 2) PCF 산정 화면의 핵심 개념 설계
2-1. “제품 단독” 조회는 불충분

같은 제품이어도 공급망 구조(협력사 라인업)가 바뀌면 PCF 값이 달라진다.
따라서 PCF 산정 단위는:

✅ “제품 + 공급망 구성(버전)” = PCF 산정 대상 (Cluster / Supply Chain Version)

이 단위를 UI상에서 명확히 정의한다.

예시 명칭:

공급망 버전

Supply Chain Version

Cluster(제품 기준 공급망 묶음)

✅ 3) PCF 산정 탭 IA/레이아웃 (데이터 조회/관리와 동일한 패턴)

PCF 산정 화면은 기존 화면들처럼 “조회 조건”이 상단에 존재하고
하단에 “산정 대상 리스트 + 상세” 구조로 설계한다.

3-1. 상단 조회 조건(필터) 설계

조회 조건은 다음 드롭다운(또는 검색형 선택)을 포함한다:

▽ 고객사 (ALL 포함)

▽ 프로젝트 (ALL 포함)

▽ 제품 (ALL 포함)

▽ 공급망 버전(Cluster/Supply Chain Version) (ALL 포함)

표시 형태 예:

SCV-2026-01 | Battery Module A | Tier1: 3개 | Tier2: 7개 | v1.2

▽ 기간 (월/분기 혼합 조회 지원)

기본은 월 단위 선택 UI

▽ 산정 상태 (미산정 / 산정중 / 산정완료 / 검토요청 / 반려)

버튼:

[산정 대상 조회]

[초기화]

[저장된 조건]

3-2. 산정 대상 리스트(하단 카드/테이블)

조회 후 결과 리스트에는 “제품 + 공급망 버전” 단위의 행이 나열된다.

각 행 컬럼 예시:

공급망 버전 ID (SCV-xxxx)

제품명

고객사/프로젝트

적용 기간

협력사 수(차수별)

데이터 커버리지(%) (필수 항목 입력 완료율)

DQR 평균

산정 상태

[상세보기] 버튼

[PCF 산정 실행] 버튼 (상태가 조건 충족 시 활성화)

✅ 4) PCF 산정 상세 화면(가장 중요)

“상세보기”를 누르면 모달/드로어/상세 페이지로 이동한다.
여기서 최종 산정에 필요한 데이터를 통합적으로 검토하고 실행한다.

4-1. 상세 화면 상단 요약 (Summary)

제품명 + 공급망 버전명

적용 기간

산정 결과(kgCO2e / 기능단위) (산정 전에는 “—”)

데이터 커버리지 %

DQR 요약

[PCF 산정 실행] / [재산정] / [결과 확정] 버튼

4-2. 상세 화면 탭 구성 (3개 추천)
탭 A) 공급망 데이터 매핑 (Data Mapping)

Tier 트리/계층형 테이블(접기/펼치기)

각 협력사별 제출 데이터 요약(분기값 + 월추정치)

필수 필드 누락 표시 (누락 시 붉은 배지)

데이터 소스 구분:

협력사 제출

원청 내부 데이터

시스템 추정치(월 추정)

탭 B) 배출계수(EF) 추적 (EF Traceability) (핵심 탭)

각 협력사 / 각 항목별로 어떤 배출계수를 썼는지 한눈에 보여준다.

테이블 컬럼 예시:

Tier / 회사명 / 사업장

데이터 항목 (전력, LNG, 소재 투입, 운송 등)

활동량 값 + 단위

EF ID

EF 출처/DB (Internal DB, ecoinvent, DEFRA, EPA 등)

적용 국가/연도

계산된 배출량(kgCO2e)

[EF 변경] 버튼(권한 있는 사용자만)

변경 이력(툴팁/모달)

탭 C) 산정 결과 및 감사 추적 (Result & Audit)

최종 PCF 결과 요약

Scope별 기여도

협력사별 기여도 랭킹

변경 로그(누가 언제 EF/값 수정했는지)

증빙 파일 링크(계약서/검증서/첨부문서)

✅ 5) 공급망 버전(Cluster) 생성/관리 UX (PCF 산정 탭 내부 또는 연결)

공급망 버전이 없으면 PCF 산정이 불가능하므로
PCF 산정 탭 내에서 다음 CTA 제공:

[공급망 버전 생성]

제품 선택

기준 Tier1 선택(또는 공급망 맵 기반)

포함할 협력사 트리 확정

버전명/태그 입력

생성 후 “SCV-xxxx” 발급

또는 기존 “공급망 보기” 모식도 화면에서
[이 구조를 공급망 버전으로 저장] 기능 제공

✅ 6) 디자인 톤/일관성

기존 AIFIX UI 스타일 유지 (라운드 카드, 보라색 CTA, 얇은 그레이 라인)

기존 “데이터 조회/데이터 관리”의 패턴을 그대로 재사용하여 학습 비용 최소화

계층형(티어) 데이터는 토글/아코디언 UI 사용

EF 추적 탭에서는 “EF ID”와 “출처”를 강조 컬럼으로 시각적으로 강조

✅ 7) 더미 데이터 요구사항

최소 3개의 공급망 버전(SCV) 더미 생성

같은 제품이라도 공급망 버전이 다르면 협력사 구성이 다르게 보이도록

EF 추적 탭에서 협력사별로 서로 다른 EF 출처/연도/국가가 섞여 있도록 구성

원하면 다음 단계로, 공급망 버전(Cluster/SCV) 명명 규칙까지 딱 정해줄게.
(예: SCV-{고객사코드}-{제품코드}-{YYYY}-{rev} + “변경 사유” 필드 포함)
