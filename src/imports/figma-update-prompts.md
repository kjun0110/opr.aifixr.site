✅ 2️⃣ 자동 매칭임을 명시해야 함

지금 구조는 자동이지만
사용자는 그걸 모를 수 있어.

이걸 안 보여주면:

사용자가 공급망 바뀐 걸 인지 못함

감사 대응 시 설명 어려움

🔥 피그마 수정 프롬프트 ②
📌 자동 매칭 배지 추가

적용 공급망 정보 바에
“자동 매칭” 배지를 표시하라.

사용자가 수동으로 변경하면
배지는 “수동 선택”으로 변경된다.

✅ 3️⃣ 오버라이드 UI는 “모달”로만

중요한 점 👇

공급망 드롭다운을 조회조건에 넣지 마.

대신:

[다른 버전 선택] 클릭 시 모달 오픈

모달 내용:

동일 제품의 공급망 그룹 목록

각 그룹의 버전 목록

구조 변경 여부

기준일

선택 시:

화면 상단 배지가 “수동 선택”으로 변경

🔥 피그마 수정 프롬프트 ③
📌 공급망 버전 선택 모달 추가

데이터 조회 화면에서
공급망을 직접 드롭다운으로 선택하지 않는다.

대신 “적용 공급망 정보 바”의
[다른 버전 선택] 버튼 클릭 시
모달 창을 통해 버전 선택하도록 설계한다.

모달 구성:

| 그룹 | 버전 | 기준일 | 구조 변경 여부 | 상태 |

❗ 추가로 건드릴 필요 없는 것

결과 리스트 구조도 유지

🎯 왜 이 정도만 하면 되냐?

우리는 지금

공급망은 자동 매칭
필요 시 오버라이드

라는 UX 철학을 유지하려는 거야.

그래서 “선택”을 기본에 노출시키지 않고
“통제권”만 제공하는 게 핵심이야.
