AIFIX – PCF 계산 트리(Computation Tree) 기반 상세 페이지 설계

너는 제조업 대기업 대상 ESG/공급망 SaaS AIFIX의 시니어 프로덕트 디자이너다.

현재 PCF 산정 화면의 “상세보기”는 단순 요약 모달 수준이다.
이를 규제 대응 및 감사 추적이 가능한 **전용 상세 페이지(Full Audit View)**로 재설계하라.

✅ 0️⃣ 기본 원칙

상세보기는 모달이 아니라 독립 페이지로 설계

계산 투명성을 최우선으로 한다

“결과 숫자”가 아니라 “계산 구조”를 보여준다

트리 기반 구조(Computation Tree)로 설계

✅ 1️⃣ 페이지 상단 – 계산 메타 정보 카드

상단 고정 영역에 다음 정보를 카드 형태로 배치:

PCF 계산 ID

프로젝트명

제품명

적용 공급망 버전

적용 M-BOM 버전

적용 기간

계산일

총 배출량 (kgCO₂e)

전체 커버리지 (%)

평균 DQR

계산 상태

레이아웃: 2열 그리드 카드 형태

✅ 2️⃣ 핵심 영역 – 계산 트리 (Computation Tree)

페이지 중앙에 트리 기반 구조 설계.

트리 구조 예시:

총 PCF (32,420 kgCO₂e)
 ├ 원청 (8,240)
 │   ├ 전력 사용
 │   ├ LNG 사용
 │   ├ 공정 배출
 │   └ 사내 물류
 ├ Tier1 – 한국배터리 (6,200)
 │   ├ 리튬 원료
 │   ├ 음극재 가공
 │   └ 전력 사용
 └ Tier2 – 셀테크 (3,100)
     ├ 리튬 채굴
     └ 운송
🔵 트리 UI 설계 요구사항

계층 접기/펼치기 기능

각 노드에 기여 배출량 표시

각 노드에 기여 비율 표시 (%)

색상은 단계별로 구분 (원청 / Tier1 / Tier2)

✅ 3️⃣ 항목 레벨 상세 테이블

각 Emission Item(배출 항목)을 펼치면 다음 테이블 표시:

| 항목 | 활동 데이터 | 단위 | 배출계수(EF) | EF 출처 | 계산 결과 |

예시:

| 전력 사용 | 5,000 | kWh | 0.459 | KR Grid 2026 | 1,836 |

✅ 4️⃣ 계산식 표시 기능

각 항목 우측에 “수식 아이콘” 배치

클릭 시 툴팁 또는 인라인 확장:

배출량 = 활동데이터 × EF × BOM 비율
5,000 × 0.459 × 0.8 = 1,836 kgCO₂e

계산식은 실제 숫자로 보여줄 것.

✅ 5️⃣ 배출계수(EF) 사용 현황 섹션

페이지 하단에 EF 사용 테이블 추가:

| EF명 | 값 | 버전 | 출처 | 데이터 유형 |

데이터 유형 구분:

1차 데이터 (협력사 제출)

2차 데이터 (LCI DB)

국가 평균

내부 계산

✅ 6️⃣ 데이터 품질 요약 영역

별도 카드 영역으로 구성:

1차 데이터 비율

2차 데이터 비율

Tier별 커버리지

평균 DQR

데이터 공백 항목 수

시각화: 도넛 차트 또는 막대 그래프

✅ 7️⃣ 계산 로그 이력

페이지 하단에:

계산 실행 일시

실행자

적용 버전 기록

재계산 여부

감사 대응용 로그 테이블 구성.
