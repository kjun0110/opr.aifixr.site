AIFIX – 데이터 조회 탭 직무 분리 + Tier 0 고정 구조 반영

너는 제조업 공급망 ESG/PCF SaaS AIFIX의 시니어 프로덕트 디자이너다.

현재 “데이터 조회” 화면은 구매직무와 PCF직무의 데이터가 혼합되어 있다.
이를 직무 기준으로 명확히 분리하되, 두 세부 탭 모두 반드시 원청(Tier 0)을 포함하도록 설계하라.

🚨 3️⃣ 가장 중요한 설계 조건 – Tier 0 (원청) 포함

두 세부 탭 모두 결과 리스트 최상단에 “Tier 0 (원청사)”를 반드시 포함하라.

원청은 항상 고정 노드이다.

협력사와 동일한 Row 구조를 가지되,

직무에 맞는 데이터만 표시한다.

원청은 collapse 불가.

항상 펼쳐진 상태 유지.
