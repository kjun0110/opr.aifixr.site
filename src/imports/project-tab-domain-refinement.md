AIFIX – 프로젝트 관리 탭 도메인 정합성 최종 고도화

너는 제조업 대기업 대상 ESG/공급망 관리 SaaS AIFIX의 시니어 프로덕트 디자이너다.

현재 프로젝트 관리 탭은 기본 구조는 완성되었으나,
공급망/제품/프로젝트 간 도메인 관계가 UI에 완전히 반영되지 않았다.

다음 사항에 맞게 프로젝트 관리 탭 내부 UI를 수정하라.

✅ 1️⃣ 프로젝트 목록 화면 수정 (중요)
🎯 문제

현재 리스트에서
“적용 공급망 버전”이 프로젝트 단일 값처럼 보인다.

하지만 공급망은 제품 단위로 연결된다.

→ 도메인 논리 충돌 발생.

🔥 수정 지시
① 컬럼 구조 변경

다음 컬럼 제거:

적용 공급망 버전

적용 M-BOM 버전

대신 다음으로 변경:

연결 제품 수

구조 미연결 제품 수

M-BOM 미선택 제품 수

PCF 계산 완료 제품 수

② 프로젝트 상태 로직 명확화

프로젝트 상태는 “제품 연결 상태 기반”으로 표시한다.

상태 체계:

제품 미연결

구조 미연결 제품 존재

M-BOM 미선택 제품 존재

PCF 일부 완료

PCF 완료

프로젝트 단일값 기반이 아니라
제품 집계 기반으로 표현한다.

✅ 2️⃣ 제품 연결 UX 재설계 (핵심 수정)
🎯 문제

현재 “제품 연결” 모달은 제품 선택만 가능하다.

공급망 / M-BOM은 이후 화면에서 설정되는 구조로 보인다.

→ UX 분리 → 실무 혼란 가능.

🔥 수정 지시

제품 연결 모달을 3단계 연결 모달로 변경하라.

Step 1️⃣ 제품 선택

제품 드롭다운

Step 2️⃣ 공급망 선택

해당 제품의 공급망 그룹 리스트 표시

그룹 선택

버전 선택

구조 변경 여부 표시

기준일 표시

Step 3️⃣ M-BOM 선택

해당 제품의 M-BOM 리스트

유효기간 표시

버전 표시

완료 버튼 조건

제품

공급망

M-BOM

3개 모두 선택되어야 완료 가능.

✅ 3️⃣ 프로젝트 상세 화면 상태 로직 정리
🎯 문제

“구조 미연결” 상태 의미가 불명확.

🔥 수정 지시

각 제품 카드에 다음 상태 표시 명확화:

공급망 미선택

M-BOM 미선택

PCF 미계산

PCF 완료

프로젝트 상단 상태는
제품 상태 집계 기반으로 계산한다.

✅ 4️⃣ PCF 실행 버튼 조건 강화

공급망 미선택 → 비활성화

M-BOM 미선택 → 비활성화

툴팁 표시:

“공급망과 M-BOM이 모두 선택되어야 PCF 계산이 가능합니다.”

✅ 5️⃣ 도메인 설명 강화 (상단 안내 카드 수정)

현재 설명을 다음으로 정제:

프로젝트는 고객사 납품 맥락을 정의하며,
제품별로 공급망 버전과 M-BOM을 선택합니다.
공급망 구조 수정은 공급망 관리 탭에서 수행합니다.

✅ 6️⃣ 리스트 하단 통계 카드 수정

현재:

전체 프로젝트

진행 중

완료

구조 미연결

→ 다음으로 변경:

전체 프로젝트

제품 미연결 프로젝트

구조 미완료 제품 수

PCF 완료 제품 수

“프로젝트 중심 통계”에서
“제품 연결 상태 중심 통계”로 변경.

✅ 7️⃣ 삭제/수정 UX 명확화

제품 카드에서:

✏ 수정 → 공급망/M-BOM 변경

🗑 삭제 → 제품 연결 해제

확인 모달 추가:

해당 제품 연결을 삭제하시겠습니까?
PCF 결과도 함께 삭제됩니다.

🎯 최종 목표

이 수정 작업을 통해:

공급망 = 1계층 자산

프로젝트 = 2계층 맥락

제품 = 연결 단위

PCF = 실행 단위

라는 도메인 구조가
UI에 완전히 일치하도록 만든다.
