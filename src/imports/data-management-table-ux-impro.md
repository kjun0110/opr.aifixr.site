AIFIX 데이터 관리 탭 – 조회 결과 테이블 구조 단순화 및 UX 개선

현재 데이터 관리 탭의 조회 결과 테이블은 다음 문제가 발생하고 있다.

1️⃣ 컬럼 수가 너무 많아 테이블 가로 폭이 과도하게 넓어졌다.
2️⃣ 프로젝트마다 가로 스크롤이 생겨 UX가 복잡해졌다.
3️⃣ 공급 데이터와 탄소 데이터가 하나의 테이블에 과도하게 섞여 가독성이 떨어진다.

따라서 조회 결과 테이블 구조를 단순화하여 가독성과 UX를 개선하라.

1️⃣ 프로젝트 구조는 유지

현재 구조는 유지한다.

프로젝트 카드
→ 클릭 시
→ 협력사 Row 표시

예

SCN-2026-01
BMW iX5 배터리 공급

SCN-2026-02
Mercedes EQS 차세대 배터리

SCN-2026-03
Audi Q6 e-tron ESS

프로젝트는 Accordion 구조를 유지한다.

2️⃣ 프로젝트별 가로 스크롤 제거

현재는 프로젝트마다 테이블 가로 스크롤이 존재한다.

이 구조를 제거한다.

변경 원칙

페이지 전체에서
가로 스크롤은 최대 1개만 존재하도록 한다.

프로젝트별 가로 스크롤은 사용하지 않는다.

6️⃣ 상세보기로 이동할 데이터

다음 데이터는 테이블에서 제거하고 상세보기 모달로 이동한다.

원재료 투입량

즉 테이블은 요약 데이터, 상세보기는 세부 데이터 구조로 만든다.

이 계층 구조는 유지한다.

Row 확장 기능을 통해 공급망 구조를 표시한다.

9️⃣ UX 목표

조회 결과 테이블은 다음 원칙을 따른다.

한 Row에서 핵심 정보만 확인 가능

세부 데이터는 상세보기에서 확인

가로 스크롤 최소화

공급망 계층 구조 유지

10️⃣ 디자인 원칙

이 테이블은 데이터 요약 대시보드 역할을 한다.

따라서

조회 화면에서는 핵심 데이터만 보여주고
세부 데이터는 상세 화면에서 제공하는 구조로 설계한다.
