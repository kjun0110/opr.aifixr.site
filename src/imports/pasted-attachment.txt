좋아.
지금 이건 “자사 데이터 관리”를 제대로 된 운영 화면으로 재정의하는 작업이야.
구조를 한 단계 정리할 타이밍이 맞아.

핵심은 두 가지야:

1️⃣ 자사 데이터 관리도 “조회조건 + 결과 영역” 구조로 간다
2️⃣ 구매관점 / PCF관점은 조회조건은 일부 공통, 결과 컬럼은 완전히 다르게 간다

그리고 디자인 틀은
👉 네가 말한 네번째 사진 구조(협력사 데이터 관리용 틀) 로 통일한다.

AIFIX – 자사 데이터 관리 화면 전면 개편

너는 제조업 ESG·공급망 SaaS AIFIX의 시니어 프로덕트 디자이너다.

현재 “자사 데이터 관리” 화면은 관점별로 테이블만 다른 구조이다.
이를 협력사 데이터 관리 화면과 동일한 UI 틀로 재설계하라.

✅ 1️⃣ 전체 구조 통일

화면 구조는 다음과 같이 구성한다:

[ 조회 조건 카드 영역 ]
[ 자사 데이터 테이블 영역 ]

협력사 데이터 관리 화면과 동일한 디자인 틀을 사용한다.

✅ 2️⃣ 조회 조건 (공통 구조)

조회 조건 카드 안에 다음 드롭다운을 배치한다:

공통 드롭다운:

▽ 기간 (월 단위, YYYY-MM)

▽ 사업장

▽ 공장

▽ 공정

버튼:

[조회]

[초기화]

🔵 3️⃣ 구매관점 – 자사 데이터 관리

조회조건 추가:

▽ 데이터 상태 (임시저장 / 확정 / 검토완료)

테이블 컬럼:

| 기간 | 사업장 | 공장 | 공정 | 생산량 | 납품량 | 가동률 | 입력 상태 | 수정 |

상단 버튼:

엑셀 다운로드

엑셀 업로드

저장

🟣 4️⃣ PCF 관점 – 자사 데이터 관리

조회조건 추가:

▽ 산정 상태 (계산완료 / 미계산 / 검토중)

테이블 컬럼:

| 기간 | 사업장 | 공장 | 공정 | 전력(kWh) | LNG(m³) | Scope1 | Scope2 | DQR | 계산 가능 | 수정 |

✅ 5️⃣ 디자인 기준

협력사 데이터 관리와 동일한 카드형 조회 조건

동일한 테이블 UI

동일한 버튼 스타일

관점 전환 시 컬럼만 변경

🎯 설계 의도

자사 데이터 관리와 협력사 데이터 관리의 UI 일관성 확보

월 단위 입력 기반 구조 정립

프로젝트/제품과 무관한 내부 운영 데이터 구조 확립

관점별 컬럼 분리로 직무 집중도 향상

💡 이 구조의 장점

✔ 원청 내부 운영 데이터 구조가 명확해짐
✔ 협력사 화면과 UX 통일
✔ 직무 전환 시 혼란 최소화
✔ 발표 시 “구조 일관성” 어필 가능

이제 이 구조로 가면
전체 시스템 IA가 안정된다.
