너는 제조업 공급망 ESG/PCF 관리 SaaS **AIFIX**의 시니어 프로덕트 디자이너다.
현재 “데이터 조회” 화면의 UI 구조를 수정하라.

이 화면은 프로젝트와 제품 단위로 PCF 데이터 조회 결과를 보여주는 화면이다.
현재 상단에 **보라색 카드 형태로 “적용 공급망 (SC-1 등)”을 표시하는 영역이 존재한다.**

그러나 실제 데이터 구조에서는 **공급망이 프로젝트/제품마다 각각 다르게 적용되므로**
상단에 단일 공급망이 적용된 것처럼 보이게 하는 UI는 **데이터 구조와 맞지 않아 사용자에게 혼동을 준다.**

따라서 아래와 같이 UI 구조를 수정하라.

---

1. 상단 “적용 공급망” 보라색 카드 삭제

현재 조회 결과 상단에 있는
“적용 공급망 : SC-1” 형태의 보라색 카드를 **완전히 제거한다.**

이 정보는 전체 조회 결과의 공통 정보가 아니라
각 프로젝트/제품에 개별적으로 적용되는 정보이므로
상단 공통 카드로 표현하지 않는다.

---

2. 공급망 정보 위치 변경

각 **프로젝트 카드 내부**에 공급망 정보를 표시하도록 구조를 변경한다.

예시 구조

프로젝트 카드

SCN-2026-01
BMW iX5 배터리 공급

제품 : 배터리 모듈 A
고객사 : BMW
기간 : 2026-01

공급망 : SC-1

[공급망 보기]

---

3. 공급망 버전이 여러 개인 경우

조회 기간이 길어지면 하나의 프로젝트/제품에도
여러 공급망 버전이 적용될 수 있다.

예

SC-1
SC-2
SC-3

이 경우 버튼을 여러 개 만들지 말고 다음 방식으로 표시한다.

공급망 : 3개

[공급망 목록 보기]

버튼을 누르면
해당 프로젝트에 적용된 공급망 버전 리스트가 표시되는 구조로 설계한다.

---

4. 버튼 구조 규칙

공급망이 하나일 경우

공급망 : SC-1
[공급망 보기]

공급망이 여러 개인 경우

공급망 : 3개
[공급망 목록 보기]

버튼은 프로젝트 카드 우측 영역에 배치한다.

---

5. UI 계층 구조 유지

현재 화면 구조는 다음과 같은 계층을 가진다.

조회 조건
→ 프로젝트 카드 리스트
→ 프로젝트 내부 정보

이 계층 구조는 유지하되
공급망 정보는 **프로젝트 카드 내부 메타 정보 영역**에 포함시킨다.

---

6. UX 원칙

이 화면의 목적은 **PCF 데이터 조회 결과를 빠르게 확인하는 것**이다.

따라서

* 공급망 구조 전체를 바로 보여주지 않는다
* 필요할 때만 “공급망 보기”로 진입하도록 한다
* 결과 리스트의 가독성을 우선한다

---

7. 최종 목표

데이터 조회 화면에서

“조회 결과 전체에 하나의 공급망이 적용된 것처럼 보이는 UI 문제”를 제거하고
“프로젝트 단위 공급망 적용 구조”를 명확히 보여주는 UI로 수정한다.
