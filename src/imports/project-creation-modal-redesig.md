너는 제조업 공급망 ESG/PCF 관리 SaaS **AIFIX**의 시니어 프로덕트 디자이너다.

현재 “프로젝트 관리” 화면에서 **[프로젝트 생성] 버튼을 누르면 나타나는 프로젝트 생성 모달**의 UI를 수정하라.

이 시스템은 기업 내부 **SRM / ERP 원천 시스템과 인터페이스되어 프로젝트 정보가 자동으로 연동되는 구조**이므로,
사용자가 프로젝트 정보를 직접 입력하는 방식이 아니라 **원천 시스템 데이터를 선택하는 방식의 UI**로 변경해야 한다.

따라서 아래 기준에 따라 UI를 수정하라.

---

1. 입력 필드를 드롭다운 선택 방식으로 변경

현재 다음 항목들은 텍스트 입력 필드로 되어 있다.

* 프로젝트 코드
* 프로젝트명
* 고객사
* 납품 공장명
* 법인명
* 생산 거점

이 항목들은 모두 **SRM / ERP에서 인터페이스된 데이터 목록을 선택하는 방식**이므로
**텍스트 입력 필드를 제거하고 드롭다운 선택 UI로 변경한다.**

예시

프로젝트 코드
[ SCN-2026-01 ▼ ]

프로젝트명
[ BMW iX5 배터리 공급 ▼ ]

고객사 선택
[ BMW ▼ ]

---

2. 프로젝트 선택 → 정보 자동 채움 구조 적용

사용자가 **프로젝트 코드 또는 프로젝트명을 선택하면**
다음 정보가 **자동으로 채워지는 구조로 설계한다.**

자동 채움 대상

* 고객사
* 납품 공장명
* 법인
* 생산 거점
* 납품 지역

예시 UX

프로젝트 코드 선택
SCN-2026-01 선택

자동 표시

고객사 : BMW
납품 공장 : Munich Plant
법인 : BMW AG
생산 거점 : Ulsan Factory

이 항목들은 **읽기 전용(Read-only) 필드 형태**로 표시한다.

---

3. 납품 기간은 사용자 입력 유지

다음 항목은 시스템 데이터가 아닌 계약 기간 설정이므로
사용자가 입력할 수 있도록 유지한다.

* 납품 시작일
* 납품 종료일

UI는 현재처럼 **Date Picker 형식**을 유지한다.

---

4. 인터페이스 데이터임을 UI에서 명확히 표시

프로젝트 정보 영역 상단에 다음 안내 문구를 추가한다.

“프로젝트 정보는 원천 시스템(SRM/ERP)에서 연동된 데이터입니다.
프로젝트를 선택하면 관련 정보가 자동으로 입력됩니다.”

이 문구는 **정보 안내형 Info Banner** 스타일로 표시한다.

---

5. 필드 구조 단순화

프로젝트 생성 모달의 입력 순서는 다음과 같이 재구성한다.

프로젝트 선택 영역

프로젝트 코드
프로젝트명

자동 표시 영역 (Read-only)

고객사
납품 공장
법인
생산 거점
납품 지역

사용자 입력 영역

납품 시작일
납품 종료일

---

6. UX 원칙

이 모달의 목적은 **사용자가 프로젝트 정보를 입력하는 것이 아니라
원천 시스템에 존재하는 프로젝트를 선택하여 AIFIX 시스템에 연결하는 것**이다.

따라서

* 텍스트 입력을 최소화한다
* 선택 기반 UI로 변경한다
* 자동 채움 방식으로 사용자 입력 부담을 줄인다

---

7. 최종 목표

“프로젝트 생성 모달”을
**수동 입력형 폼 → 원천 시스템 데이터 선택형 인터페이스 UI**로 재설계한다.
