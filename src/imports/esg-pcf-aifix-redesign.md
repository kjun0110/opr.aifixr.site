너는 제조업 ESG/공급망 PCF SaaS AIFIX의 시니어 프로덕트 디자이너다.

현재 “데이터 조회” 탭은:

조회 조건에 ALL 선택지가 없음

조회 결과가 하나의 더미 시나리오만 표시됨

계층 구조가 플랫하게 나열되어 있음

우측 점 세개 버튼이 기능 없이 존재함

이를 실제 B2B 시스템처럼 보이도록 구조적으로 개편하라.

이 개편은 반드시:

🔵 공급망 데이터 조회 탭

🟣 PCF 데이터 조회 탭

두 화면 모두 동일한 구조 논리로 적용하라.
(데이터 내용은 달라도 UX 구조는 동일)

✅ 1️⃣ 조회 조건 영역 개편
📌 모든 드롭다운에 ALL 옵션 추가

다음 항목에 모두 최상단에 추가:

고객사

프로젝트

제품

사업장

공장

공정

협력사 차수

협력사 회사명

기간 (시작/종료)

표기 방식:

ALL (전체) 를 항상 최상단에 배치

기본 선택값은:

→ ALL

✅ 2️⃣ 조회 결과 구조 개편 (다중 시나리오 생성)

현재는 하나의 공급망만 보인다.
이를 최소 3개의 원청사 기준 시나리오로 구성하라.

예시 더미 구성:

🟦 시나리오 A

AIFIX Corp. (OEM)

🟦 시나리오 B

K-Battery Global (OEM)

🟦 시나리오 C

NextCell Energy (OEM)

각 시나리오는 내부 값이 다르게 구성되어야 한다:

국가

인증상태

리스크 등급

연결 제품 수

FEOC / RMI 상태

인증 만료일

모든 값이 동일하게 반복되면 안 된다.

✅ 3️⃣ 계층형 토글 구조 적용

조회 결과를 “플랫 리스트”가 아니라
원청사 기준 Accordion 구조로 변경하라.

🔽 기본 접힘 상태

화면에는 원청사 3개만 보인다:

Tier 0 – AIFIX Corp.

Tier 0 – K-Battery Global

Tier 0 – NextCell Energy

🔽 원청사 토글 확장 시

각 원청사 아래에:

🔹 최소 구성 조건

Tier 1: 2개 이상

Tier 2: 3개 이상

Tier 3: 각 Tier2마다 최소 1개씩 연결

계층은 반드시 트리 구조로 연결:

Tier 0
 ├─ Tier 1 (2개 이상)
 │    ├─ Tier 2 (3개 이상)
 │    │    └─ Tier 3 (각각 1개 이상)
📌 표시 방식

계층별 들여쓰기 적용

Tier 배지는 색상 유지

각 계층은 독립 토글 가능

하위 노드가 있으면 화살표 아이콘 표시

✅ 4️⃣ 불필요한 점 세개 버튼 삭제

현재 각 라인 우측의 점 세개 버튼은 기능이 없다.

→ 완전 삭제하라.

✅ 5️⃣ 원청사 라인에만 "공급망 보기" 버튼 추가

Tier 0 (원청사) 라인에만 아래 버튼 추가:

🔷 버튼명:

공급망 보기

위치:
상세보기 버튼 옆

✅ 6️⃣ 공급망 보기 클릭 시 동작

클릭하면 우측 슬라이드 패널 또는 모달로
공급망 MAP 화면 표시

구성은 세번째 이미지 구조처럼:

Tier1 → Tier2 → Tier3 노드 카드

국가 표시

인증 상태 표시

리스크 등급 배지

연결선 점선 표현

닫기 버튼 포함

✅ 7️⃣ 공급망 데이터 조회 + PCF 데이터 조회 공통 적용

이 구조는:

🔵 공급망 데이터 조회

🟣 PCF 데이터 조회

두 탭에 동일하게 적용한다.

차이점:

공급망 데이터 조회:
→ 공급망 인증/리스크 중심

PCF 데이터 조회:
→ 배출량 / Scope / LCA 단계 중심

그러나:

ALL 드롭다운

원청사 기준 토글 구조

공급망 보기 버튼

계층 확장 로직

은 동일하게 유지하라.

✅ 8️⃣ UX 디테일

확장 시 부드러운 애니메이션
