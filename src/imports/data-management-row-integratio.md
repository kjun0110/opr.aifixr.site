AIFIX 데이터 관리 탭 – 조회 결과 Row 구조 통합 및 회사 유형 정의 수정

현재 데이터 관리 탭에서
조회 조건 영역은 구매 관점과 PCF 관점이 통합되었다.

하지만 조회 결과 영역의 Row 구조는 여전히 서로 다르게 표시되고 있다.

현재 문제:

구매 관점 Row
→ 회사 정보 중심

PCF 관점 Row
→ 탄소 데이터 중심

즉 두 관점이 서로 다른 컬럼 구조를 사용하고 있다.

하지만 시스템 설계 원칙은 다음과 같다.

구매 직무와 PCF 직무는 동일한 데이터를 조회해야 하며
차이는 권한(수정 가능 여부)만 존재해야 한다.

따라서 조회 결과 Row 구조를 새로 정의하여 완전히 통합하라.

1️⃣ 프로젝트 Accordion 구조 유지

현재 조회 결과의 프로젝트 그룹 구조는 그대로 유지한다.

구조

프로젝트 카드
→ 클릭 시
→ 협력사 Row 표시

예

SCN-2026-01
BMW iX5 배터리 공급

SCN-2026-02
Mercedes EQS 차세대 배터리

이 프로젝트 단위 그룹핑 구조는 유지한다.

3️⃣ 회사 유형 정의 수정 (중요)

현재 회사 유형이 다음과 같이 표현되어 있다.

OEM
Manufacturer
Supplier

하지만 OEM은 산업별로 의미가 달라
시스템의 표준 회사 유형으로 적합하지 않다.

따라서 회사 유형을 다음 3가지로 정의한다.

회사 유형

Operator
Supplier
Customer

설명

Operator
→ 시스템을 운영하는 원청사

Supplier
→ 공급망에 참여하는 협력사 (Tier1, Tier2, Tier3)

Customer
→ 원청사의 고객사 (예: BMW, Mercedes 등)

따라서 Row에서 OEM / Manufacturer 표기는 제거하고
회사 유형을 다음 값으로 표시한다.

Operator / Supplier / Customer
