너는 B2B 제조업 ESG·공급망 관리 SaaS AIFIX의 시니어 프로덕트 디자이너다.
이번 작업은 원청사 “구매직무” 전용 화면이며, 메인 네비게이션 중 [데이터 관리] 탭 화면을 개선 설계한다.
참고로 기존 화면에는 상단 탭 토글 **[자사데이터 관리] / [협력사 데이터 관리]**가 존재한다. 이 구조는 유지하되, 각 탭 내부를 **[데이터 조회] 화면과 유사한 패턴(조회조건 카드 + 테이블 + 액션바)**로 바꿔라.

0) 변경 목표 (Why)

현재 [데이터 관리] 화면은 테이블만 덩그러니 있고 “어떤 조건의 데이터를 관리하는지(스키마/컨텍스트)”가 약하다.
이를 [데이터 조회] 화면과 동일한 UX 패턴으로 통일한다:

상단에 **“입력/관리 조건(필터) 카드”**를 배치하여, 사용자가 고객사/프로젝트/제품/기간/사업장/공장/공정을 먼저 선택하게 한다.

선택 조건에 따라 아래 테이블이 동적으로 바뀌고, 그 테이블의 **스키마(표 컬럼/편집 가능한 필드)**를 토글(세그먼트)로 전환할 수 있게 한다.

엑셀 다운로드/업로드/저장/검증 등 운영 액션은 테이블 상단 컨트롤 바로 정리한다.

1) 페이지 레이아웃 (데스크톱 1440px)

상단 고정 네비게이션은 이미 존재한다고 가정하고, 본문만 설계한다.
본문은 탭 토글 아래에 2단 구조를 공통 적용하라:

입력/관리 조건 카드(상단) – 데이터 조회 화면의 “조회 조건” 카드와 동일한 스타일/그리드

결과 테이블 영역(하단) – 상단 컨트롤 바 + 테이블(인라인 편집 가능) + 상태/에러 UI

2) [데이터 관리] 공통: “입력/관리 조건” 카드 구성

데이터 조회 화면과 유사한 2줄 그리드로 배치하라.

2-1) 기본 드롭다운 (원청/고객사 구분)

▽ 고객사 (예: BMW, 벤츠)

▽ 프로젝트 (BMW Munich, BMW China 등)

▽ 제품

▽ 기간: 시작 년월 ~ 종료 년월 (두 개의 드롭다운 또는 Range Picker, ‘~’ 구분자 포함)

▽ 사업장

▽ 공장

▽ 공정

각 드롭다운은 placeholder/선택 태그/초기화 X 제공.

2-2) 조건 카드 하단 액션

[조회/불러오기] (Primary Gradient) → 선택 조건 기반으로 테이블 로드

[초기화] (secondary)

[저장된 조건] 드롭다운 (자주 쓰는 조합 저장/불러오기)

3) 핵심 변경: “스키마 토글(세그먼트 컨트롤)” 추가

[데이터 관리]는 단일 테이블이 아니라, 관리 목적별로 컬럼/입력필드가 달라지는 스키마가 존재한다.
따라서 테이블 상단에 스키마 토글을 배치하여, 클릭 시 테이블 컬럼이 바뀌도록 설계하라.

3-1) 자사데이터 관리 탭 – 스키마 토글 예시 (구매직무 관점)

세그먼트(토글) 버튼 3~5개 제안하고, 선택된 토글에 따라 테이블 컬럼이 변경:

BOM/자재 기준 (기본)

공정·생산 요약

에너지·유틸리티

품질/수율

첨부/증빙 문서

실제 데이터는 대부분 인터페이스로 들어오지만, 미입력 항목은 키인 편집 가능해야 함.

3-2) 협력사 데이터 관리 탭 – 스키마 토글 예시 (구매직무 관점)

협력사 프로필/규제 준수 (회사명/티어/국가/FEOC/RMI 등)

제출 상태/검증

납품 자재 요약(Material BOM 요약)

담당자/연락처(요약)

증빙 문서/첨부

“협력사 입력 원본 전체”가 아니라 구매가 확인하고 싶은 핵심 요약 필드만 보여주는 방향으로 구성.

4) 결과 테이블 영역 설계 (공통)
4-1) 테이블 상단 컨트롤 바(우측 액션)

데이터 조회 화면과 같은 패턴으로 정리하라:

검색창(키워드)

정렬 드롭다운(최신 수정/상태/협력사/기간)

[컬럼 선택] 버튼

[엑셀 다운로드] 버튼 (XLSX)

[엑셀 업로드] 버튼 (업로드 시 미리보기/검증 모달)

[저장] 버튼 (Primary 또는 Primary Outline)

※ 기존 화면의 “신규 추가”가 있다면, 스키마 토글에 따라 **[행 추가] / [항목 추가]**로 라벨을 더 명확히 하라.

4-2) 인라인 편집 + 즉시 벨리데이션(필수)

테이블 셀 편집 시 즉시 검증:

형식/자리수 오류 발생하면 셀 하단 빨간 보더 + 에러 캡션(14px)

예: “Tax ID는 10~12자리 숫자/하이픈 형식입니다.”

저장 버튼은 변경 사항이 있을 때만 활성화

4-3) 상태 표시

행 단위로 “관리 상태/제출 상태” 뱃지(예: 활성/비활성, 제출/검증요청/반려/승인)

마지막 수정일/수정자 표시

5) Empty / Loading / Error 상태(공통)

조건 미선택: “상단에서 조건을 선택하고 [조회/불러오기]를 눌러주세요.”

결과 없음: “해당 조건의 관리 데이터가 없습니다. 조건을 조정하거나 신규 추가하세요.”

로딩: 스켈레톤(테이블 + 컨트롤바)

업로드 오류: “업로드 파일의 N행에서 오류가 발생했습니다” + 오류 리스트 테이블

6) 최종 산출물

1440px 데스크톱 기준으로 [데이터 관리] 탭 1개 화면 안에 아래를 모두 포함해 설계하라:

상단 탭 토글: 자사데이터 관리 / 협력사 데이터 관리

각 탭 내부 공통: 입력/관리 조건 카드(데이터 조회와 동일 패턴)

테이블 상단: 스키마 토글(세그먼트) + 컨트롤바(검색/정렬/엑셀 다운로드·업로드/저장/컬럼선택)

인라인 편집 + 즉시 벨리데이션 UI

AIFIX 디자인 시스템을 엄격히 적용하고, 엔터프라이즈 B2B SaaS 수준으로 깔끔하게 완성하라.
