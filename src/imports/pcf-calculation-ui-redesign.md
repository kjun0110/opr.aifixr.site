너는 제조업 공급망 ESG/PCF 관리 SaaS **AIFIX**의 시니어 프로덕트 디자이너다.

현재 “PCF 산정” 화면의 UI 구조를 전면 수정하라.

현재 화면은 “조회 조건을 선택하면 바로 PCF 계산을 실행하는 구조”로 되어 있다.
그러나 PCF 계산은 단순 조회가 아니라 **제품 기준 탄소 산정 실행 프로세스**이므로
UI 구조를 다음 기준에 맞게 재설계해야 한다.

---

# 1. PCF 산정 기준 변경

PCF 산정은 **고객사 / 프로젝트 기준이 아니라 제품 기준**으로 수행된다.

따라서 다음 항목을 제거한다.

제거 대상

프로젝트 선택

---

# 2. PCF 산정 대상 선택 영역 재구성

조회 조건 영역을 **“PCF 산정 대상 선택” 영역으로 변경한다.**

선택 항목은 다음 3가지로 제한한다.

# 4. PCF 산정 실행 조건

PCF 계산은 다음 조건이 충족되어야 실행 가능하다.

조건

제품 선택 완료
공급망 선택 완료
BOM 선택 완료
원청 데이터 입력 완료
협력사 데이터 입력 완료

조건이 충족되지 않으면

PCF 계산 버튼은 **비활성화 상태**로 유지한다.

버튼 표시 예시

PCF 계산 실행 (비활성)

조건 충족 시

PCF 계산 실행 (활성)

---

# 5. 산정 대상 조회 버튼 제거

현재 화면에 있는

산정 대상 조회 버튼

은 제거한다.

PCF 계산은 조회 기능이 아니라
**조건 충족 후 실행되는 프로세스이기 때문**이다.

---

# 6. PCF 계산 실행 버튼

조건이 충족되면 다음 버튼을 활성화한다.

PCF 계산 실행

버튼 클릭 시

PCF 계산 작업이 시작된다.

# 9. M-BOM 상세 구조 영역 유지

현재 화면의

M-BOM 상세 구성

영역은 유지한다.

단, 이 영역은 **선택된 BOM 정보를 확인하는 역할**만 수행한다.

---

# 10. UX 목적

PCF 산정 화면은

단순 조회 화면이 아니라

“제품 기준 탄소 배출 계산 실행 화면”

이다.

따라서

조회 → 계산

구조가 아니라

대상 선택 → 데이터 준비 확인 → 계산 실행

구조로 재설계한다.
