현재 PCF 산정 화면에 **“데이터 준비 상태 카드”가 존재하지만,
준비 상태가 미완료일 때의 UX가 설계되어 있지 않다.**

PCF 산정은 다음 조건이 충족되어야 실행 가능하다.

* 원청 데이터 입력 완료
* 협력사 데이터 입력 완료

따라서 데이터 준비 상태 카드에 **상태별 UI를 추가로 설계하라.**

---

# 1. 준비 상태 3단계 표현

데이터 준비 상태는 다음 3가지 상태를 가진다.

준비 완료
부분 완료
준비 미완료

---

# 2. 준비 완료 상태 (현재 화면 유지)

예

원청 데이터 : 완료
협력사 데이터 : 12 / 12 완료
데이터 커버리지 : 95.5%

이 경우

PCF 계산 실행 버튼은 **활성화 상태**로 유지한다.

---

# 3. 부분 완료 상태 추가

예

원청 데이터 : 완료
협력사 데이터 : 9 / 12 입력

이 경우

PCF 계산 실행 버튼은 **비활성화 상태**로 표시한다.

그리고 카드 하단에 안내 메시지를 추가한다.

예

"협력사 데이터 입력이 완료되지 않았습니다."

---

# 4. 미완료 상태 추가

예

원청 데이터 : 미입력
협력사 데이터 : 0 / 12

이 경우

PCF 계산 실행 버튼은 비활성화 상태이며
카드 하단에 다음 안내 메시지를 표시한다.

예

"PCF 계산을 위해 원청 및 협력사 데이터 입력이 필요합니다."

---

# 5. 버튼 상태 표시

PCF 계산 실행 버튼은 다음 상태를 가진다.

활성
비활성

비활성 상태에서는 버튼 Hover 시
비활성 이유 Tooltip을 표시한다.

예

"협력사 데이터 입력이 완료되지 않았습니다."

---

# 6. UX 목적

사용자가 다음 질문을 즉시 이해할 수 있도록 한다.

* 왜 PCF 계산이 안 되는가
* 어떤 데이터가 부족한가
* 무엇을 먼저 해야 하는가
