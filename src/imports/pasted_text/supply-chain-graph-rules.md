현재 공급망 그래프에서 세부제품을 선택했을 때
일부 노드(Tier1)가 표시되지만 그 아래 Tier2 / Tier3 노드가 존재하지 않는 문제가 있다.

예시

세부제품 A''' 선택 시
Tier1 : 글로벌셀 표시
하지만 글로벌셀 아래 Tier2 / Tier3 공급사가 없음

이는 공급망 구조상 논리적으로 맞지 않는다.

공급망 그래프는 각 노드가 실제 공급 관계를 가진 하위 노드를 반드시 포함해야 한다.

따라서 다음 규칙을 적용한다.

1. 공급망 구조 표현 규칙 수정

그래프 노드 표시 기준을 다음과 같이 변경한다.

현재

BOM 포함 여부 기준 노드 표시

수정

공급망 관계 + BOM 필터 함께 적용

즉

BOM에 포함된 노드만 강조
하지만 공급망 구조는 유지
2. BOM 필터 동작 방식 수정

세부제품 선택 시

현재

해당 BOM에 포함된 노드만 표시

수정

전체 공급망 구조 유지
해당 BOM에 포함된 노드 강조
해당 BOM과 무관한 노드는 opacity 30%

그래프 구조 자체는 유지한다.

3. Tier1 노드는 최소 한 개 이상의 하위 노드를 가져야 한다

그래프 규칙

Tier1 → Tier2
Tier2 → Tier3

연결이 존재해야 한다.

따라서 예시 데이터에서도

글로벌셀
 └ Tier2 소재 공급사
      └ Tier3 원재료 공급사

구조를 추가한다.

예시 노드

글로벌셀 (Tier1)
 ├ 양극재소재 (Tier2)
 │   └ 니켈광산 (Tier3)
 └ 전해질소재 (Tier2)
     └ 리튬광산 (Tier3)
4. 그래프 데이터 예시 보강

데모 화면에서는 각 Tier1 노드 아래 최소

Tier2 2개
Tier3 1~2개

정도의 예시 공급망을 보여준다.

이렇게 해야 사용자가

공급망 구조를 직관적으로 이해할 수 있다
5. 노드 연결선 검증

그래프 연결선은

부모 → 자식

관계가 존재할 때만 표시한다.

연결선 없이 단독 노드가 나타나는 상황은 제거한다.

기대 효과

이 수정으로

공급망 구조의 논리적 일관성 확보
그래프 이해도 향상
데모 시 설명력 강화

그리고 하나 더 중요한 포인트가 있어.

지금 네 UI 구조는 사실 이 모델을 쓰고 있어:

Supply Chain Graph
+ BOM Filter
+ PCF Contribution

이건 굉장히 좋은 구조야. 다만 지금 데이터 예시가 약해서 그래.

데모용이라면 보통 이렇게 만든다:

Tier0 1개
Tier1 3개
Tier2 6개
Tier3 6~9개

그래야 그래프가 진짜 공급망처럼 보인다.
