✅ 1️⃣ 인증 관리 탭의 전체 구조

인증 관리 탭 안에는 상위 인증 카테고리 탭을 둔다.

상위 탭

RMI

FEOC

(향후 확장 가능 구조로 설계)

✅ 2️⃣ RMI 탭 설계
🎯 목적

글로벌 RMI 인증 기업 전체 리스트 관리

우리 공급망 등록 협력사와 교차 표시

구매직무가 인증 상태를 빠르게 검색/확인 가능

🔹 2-1. 상단 요약 대시보드 영역

카드형 4개:

전체 RMI 등록 기업 수

우리 협력사 중 RMI 인증 기업 수

만료 예정 기업 수 (90일 이내)

미인증 협력사 수

카드 클릭 시 자동 필터 적용

🔹 2-2. 데이터 구분 방식 (중요)

RMI 탭에서 데이터는 두 레이어로 존재해야 함:

Layer 1: 글로벌 RMI 전체 기업 리스트

전체 인증 기업

인증번호

인증 유형

유효기간

국가

인증 상태

Layer 2: 우리 공급망에 등록된 기업

👉 “우리 협력사 여부” 컬럼 추가

표시 방식:

Badge: “우리 공급망”

색상 강조

또는 체크 아이콘

🔹 2-3. 리스트 테이블 설계

컬럼 예시:

| 기업명 | 국가 | 인증번호 | 인증유형 | 유효기간 | 상태 | 우리공급망 여부 |

🔹 2-4. 우리 협력사인 경우 추가 정보 표시

방법 2가지 중 UX 적절한 방식 선택:

옵션 A (권장)

마우스 Hover 시 Tooltip 카드 표시

Tooltip 내용:

공급망 차수 (Tier 1 / 2 / 3)

해당 제품

프로젝트명

등록일

최근 제출 상태

옵션 B

행 클릭 시 우측 Slide Panel 오픈

🔹 2-5. 검색 및 필터 기능

상단 필터:

기업명 검색

국가 필터

인증상태 필터

“우리 공급망만 보기” 토글

🔹 2-6. 데이터 업데이트 구조 표시

상단 우측에 작은 정보 텍스트:

“RMI 데이터는 매월 자동 업데이트됩니다.”

최근 업데이트 일자 표시

✅ 3️⃣ FEOC 탭 설계
🎯 목적

FEOC는 공개 글로벌 리스트가 없음.
→ 우리 협력사 데이터 기반 관리 필요

🔹 3-1. 상단 요약 카드

전체 협력사 수

FEOC 해당 협력사 수

검토 필요 협력사 수

미응답 협력사 수

🔹 3-2. FEOC 판단 구조

FEOC는 협력사 자가 입력 데이터 + 내부 검토 결과 기반

리스트 테이블:

| 기업명 | Tier | 국가 | FEOC 해당 여부 | 검토 상태 | 최근 업데이트 |

🔹 3-3. FEOC 상태 배지

해당 (위험) – Red

비해당 – Green

검토중 – Yellow

정보부족 – Gray

🔹 3-4. 상세 보기 구조

기업 클릭 시:

우측 슬라이드 패널 열림

패널 내용:

회사 기본정보

FEOC 판단 근거 데이터

관련 제품

관련 프로젝트

공급망 차수

검토 이력

✅ 4️⃣ 공통 UX 설계 원칙

인증 관리 탭은 “조회 중심” + “리스크 관리 중심”

리스트 + 요약 대시보드 구조 유지

기존 AIFIX UI 톤앤매너 유지

공급망 네트워크 화면과 시각적 통일성 유지

✅ 5️⃣ RMI vs 우리 협력사 구분 전략

RMI 탭에서 다음 구조로 구분하라:

기본은 글로벌 전체 리스트

“우리 공급망만 보기” 토글 제공

우리 협력사일 경우 시각적 강조

전체 리스트와 우리 리스트를 분리된 탭으로 나누지 말 것

(하나의 테이블에서 필터 기반 전환이 더 직관적)

✅ 6️⃣ 확장성 고려

향후 다음 인증 추가 가능하도록 설계:

IRMA

LBMA

ISO 관련 인증

EU Battery Passport

탭 구조 확장 가능하게 설계

🚫 하지 말 것

RMI와 FEOC를 한 화면에 섞지 말 것

우리 협력사 리스트만 따로 고립시키지 말 것

인증 정보를 단순 CSV 업로드 관리 화면으로 만들지 말 것

단순 CRUD 관리 화면으로 설계하지 말 것
