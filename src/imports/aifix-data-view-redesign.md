AIFIX – 단일 원청 + 다중 프로젝트 조회 구조 반영

너는 제조업 ESG/PCF SaaS AIFIX의 시니어 프로덕트 디자이너다.

현재 데이터 조회 화면에서
ALL 조회 시 Tier 0가 여러 회사로 표시되는 구조가 있다.

이는 잘못된 표현이다.

AIFIX는 단일 원청 전용 시스템이므로:

Tier 0 회사는 항상 동일해야 한다.

단, 프로젝트/제품/버전/기간이 다르면 조회 결과는 여러 건 나올 수 있다.

이 구조를 반영하여 화면을 수정하라.

✅ 1️⃣ 핵심 설계 원칙
🔵 원칙 1

Tier 0 = 시스템에 등록된 단 하나의 원청사
(예: 삼성SDI)

🔵 원칙 2

ALL 조회 시 여러 줄이 나오는 이유는:

프로젝트가 다르기 때문

제품이 다르기 때문

버전이 다르기 때문

기간이 다르기 때문

회사가 다른 것이 아니다.

✅ 2️⃣ 조회 결과 구조 개편
❌ 기존 잘못된 구조
Tier 0 - AIFIX Corp.
Tier 0 - K-Battery Global
Tier 0 - NextCell Energy

→ 서로 다른 OEM처럼 보임 (오류)

✅ 수정 후 구조

Tier 0는 동일 회사지만
“프로젝트/제품” 단위로 묶여 여러 건 표시되도록 변경

📌 변경 구조

조회 결과 상단에 다음 정보를 추가:

원청사: 삼성SDI
조회 결과: 3개 프로젝트
📌 테이블 구조 변경
기존 컬럼

수정 컬럼

| 프로젝트 | 제품 | 버전 | Tier | 회사 | Scope1 | Scope2 | ... |

✅ 3️⃣ 계층 구조 표현 방식

Tier 0는 여러 개로 보이지 않도록:

방법 1 (권장)

프로젝트 단위 Accordion 구조:

▶ 프로젝트 A
   └ Tier 0 - 삼성SDI
        ├ Tier 1
        ├ Tier 2
방법 2

Tier 0는 고정 상단 표시 + 그 아래 프로젝트별 그룹

[원청사: 삼성SDI]

▶ 프로젝트 A
▶ 프로젝트 B
▶ 프로젝트 C
✅ 4️⃣ PCF 데이터 조회 화면도 동일 적용

PCF 조회 화면도:

Tier 0는 단일 회사 유지

Scope 수치가 다른 이유는 프로젝트/제품 차이 때문

회사명이 바뀌면 안 된다.

✅ 5️⃣ 공급망 데이터 조회 화면도 동일 적용

공급망 화면에서도:

Tier 0는 하나

프로젝트 단위로 공급망 구조가 달라질 수 있음

공급망 보기 버튼은:

프로젝트 행 우측에 위치

해당 프로젝트의 공급망 MAP만 보여주도록 설계

✅ 6️⃣ 더미 데이터 수정 지침

다음은 금지:

서로 다른 OEM 이름을 Tier 0에 사용

Tier 0 국가가 프로젝트마다 달라짐

대신:

회사명 동일 유지

프로젝트명, 제품명, 버전명만 다르게
